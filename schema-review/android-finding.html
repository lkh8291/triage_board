Exported PaymentActivity allows 0-amount payment bypass via deeplink

found by android-scanner-v0.2 · 2026-04-29 05:22 UTC · report RPT-2026-04-29-B

Summary

본 취약점은 PaymentActivity.kt의 onCreate()에서 amount intent extra가 검증 없이 processPayment()로 전달되며 액티비티가 BROWSABLE 필터로 exported된 이유로 0원 결제 우회가 가능한 취약점입니다. 이로 인해 공격자는 별도 권한 없이, 피해자가 악성 shopping://pay 딥링크를 클릭할 경우 무결제 상품 수령 등의 금전 피해를 발생시킬 수 있습니다.

ai_rationale

PaymentActivity가 exported=true이고 deeplink (shopping://pay)로 외부 앱·브라우저가 호출 가능. amount 파라미터가 long 변환만 거치고 0 또는 음수에 대한 검증·서버 사이드 재검증 미확인. 0원 결제 트리거 후 logcat에서 'payment success' 확인. 서버에서 final amount 재검증 여부는 별도 점검 필요.

Remediation hint remediation_hint

  1. PaymentActivity exported=false로 변경, 또는 서명 권한(protectionLevel="signature")으로 보호.
  2. deeplink 진입 시 사용자 명시적 확인 단계 추가 (결제 금액 + "OK" 버튼).
  3. 서버 사이드에서 cart 기준 최종 amount 재검증 (클라이언트 입력값 신뢰 ❌).
