Default credentials grant root telnet access on EX-CAM-2000 fleet

found by device-scanner-v0.1 · 2026-04-29 08:11 UTC · report RPT-2026-04-29-C

Summary

본 취약점은 EX-CAM-2000 firmware 2.1.7의 telnetd 로그인 서비스에서 기본 자격증명 admin:admin이 /etc/passwd·/etc/shadow에 하드코딩된 이유로 root shell 획득이 가능한 취약점입니다. 이로 인해 공격자는 TCP/23 네트워크 도달성만 확보하면, 별도 사용자 상호작용 없이 디바이스 완전 장악 및 동일 펌웨어를 운영하는 fleet ~340대로의 lateral movement를 수행할 수 있습니다.

ai_rationale

default credential admin:admin로 telnet root shell 획득 (인증 우회 없이 직접 로그인). firmware 2.1.7에서 동일 default credential 하드코딩 확인 (firmware extract /etc/shadow 분석). 동일 firmware 사용 fleet 약 340대 영향 추정. 펌웨어 업데이트 메커니즘은 별도 점검 필요.

Remediation hint remediation_hint

  1. firmware 2.1.8+ 출시 — 초기 부팅 시 default credential 강제 변경 흐름 (mandatory password setup wizard) 도입.
  2. telnet 비활성화, SSH (key-based) 전환.
  3. 기존 fleet 일괄 업데이트 + 비밀번호 강제 회전.
  4. /etc/shadow에 default credential 하드코딩 제거.
